### 关键发现
- **项目规模**: 30,193 行 Python + 32 个前端组件 + 139 个 API 路由
- **独特优势**: 自我进化引擎、多 Agent 团队(5角色)、Run&Fix循环、国产模型深度适配

- **P0 质量问题**: 核心引擎零测试覆盖(15-20%)、150+ 处 except Exception、前端长列表未虚拟化
- **P0 DevOps**: CI/CD 完全缺失(.github为空)、Node 依赖无版本锁定、版本号不一致
- **缺失功能**: 实时代码补全、编辑器分栏、调试器集成、代码格式化、内联Chat
- **前端问题**: 60+ 处 any 类型、AIPanel 879行巨石组件、Zustand 单Store膨胀、i18n 死代码

## PyCoder UI 升级 V2 — 全功能入口覆盖

### 背景
用户要求"把所有的主要功能都要有入口和图标"。前一版 ActivityBar 只有 8 项，底部面板只有"终端"暴露标签，BidTool/Cloud/Settings 等功能无直接入口。

### 改动清单（6个文件）
1. **ActivityBar.tsx** — 重构为 4 组 15 项（核心: 文件/搜索/Git/AI → AI工具: 团队/进化/技能/扩展/片段 → 实用工具: 浏览/标书/运行/命令 → 系统: 云/设置），Git 项有变更数 badge
2. **StatusBar.tsx** — 暴露全部 7 个底部面板标签（终端/输出/问题/Python运行器/测试生成/Run&Fix/调试），每个标签有图标+文字
3. **Sidebar.tsx** — 新增 bidtool 和 cloud 入口配置，BidToolPanel 组件已接入
4. **WelcomeScreen.tsx** — 快捷操作从 4 格扩展到 8 格（打开项目/AI对话/快速命令/自我进化/Agent团队/技能市场/标书制作/运行终端）
5. **CommandPalette.tsx** — 命令从 6 条扩展到 18 条，覆盖全部功能入口
6. **layout.css** — 新增 activity-group/activity-divider/activity-badge 样式，StatusBar 底部面板标签样式，ActivityBar 宽度 44→52px

### 功能入口映射
| 功能 | 入口位置 |
|------|---------|
| 文件/搜索/Git/AI | ActivityBar 核心组 |
| 团队/进化/技能/扩展/片段 | ActivityBar AI工具组 |
| 浏览器/标书/终端/命令面板 | ActivityBar 实用工具组 |
| Cloud/设置 | ActivityBar 系统组 |
| 终端/输出/问题/Python/测试/Run&Fix/调试 | StatusBar 底部面板标签 |
| 全部功能 | 命令面板 (Ctrl+K) |
