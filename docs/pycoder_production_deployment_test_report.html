🚀 部署建议

推荐部署方案

  1. Docker Compose 部署（首选）— docker-compose.yml 开箱即用
  2. 系统服务部署（备选）— systemd/supervisor 管理进程
  3. K8s 部署（扩展）— 需额外编写 Helm Chart

生产环境最小配置

# .env 生产配置
PYCODER_API_KEY=<生成强随机密钥>
PYCODER_CLOUD_JWT_SECRET=<生成强随机密钥>
DEEPSEEK_API_KEY=<你的 DeepSeek API Key>
QWEN_API_KEY=<你的 Qwen API Key>
GLM_API_KEY=<你的 GLM API Key>
PYCODER_PORT=8423
PYCODER_HOME=/data
HOST_WORKSPACE=/opt/workspace

部署命令

# 1. 配置环境
cp .env.example .env
# 编辑 .env 填入实际值

# 2. 自动修复代码格式
python -m ruff check pycoder/ --fix --unsafe-fixes
python -m black pycoder/
python -m isort pycoder/

# 3. 构建并启动
docker-compose up -d --build

# 4. 验证
curl http://localhost:8423/api/health
curl -H "X-API-Key: $PYCODER_API_KEY" http://localhost:8423/api/git/status

# 5. 查看日志
docker-compose logs -f pycoder

⚠️ 重要提醒

  • 切勿使用 PYCODER_API_KEY=disabled 在生产环境
  • 必须配置 AI 模型 API Key，否则核心功能不可用
  • 建议使用反向代理（Nginx/Caddy）提供 HTTPS 终止
  • 定期更新 AI 模型 API Key，遵循密钥轮换策略
