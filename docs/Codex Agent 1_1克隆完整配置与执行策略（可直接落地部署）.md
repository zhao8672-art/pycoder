# 二、五层官方标准架构（必须原样复刻）

## 1\. 工程感知层（Codex专属）

## 2\. 工程记忆层（四级仓库记忆体系）

Codex能搞定上万行项目，核心是工程记忆，而非普通对话记忆：

- **工程师全局记忆**：用户惯用范式、命名风格、工程模板、禁止写法、CI/CD规则

## 3\. 决策推理层（Codex 双核心推理引擎）

Codex 区别于普通代码模型的核心：**结构化代码推理 \+ 工程试错强化循环（RL 自迭代）**，并非单次生成代码，而是「生成\-运行\-报错\-修正\-复测」的自主工程闭环。

官方独有推理机制：**任务复杂度动态算力调节**，简单脚本快速输出，大型重构/跨文件迁移自动提升推理步数、深度校验、多方案比对，适配全量级工程任务。

## 4\. 沙箱执行层（Codex 安全运行核心）

所有任务隔离容器化运行，是企业级落地的关键：

- 独立云端沙箱容器，单任务独立环境、互不污染，无外网默认权限，杜绝安全风险

- 预装完整开发环境：依赖环境、运行时、编译工具、测试套件、Git 环境

- 支持终端命令执行、项目构建、单元测试、Lint 校验、版本提交全流程自动化

- 支持多任务异步并行，可同时推进功能开发、Bug 修复、代码重构、文档更新

## 5\. 闭环校验迭代层（自愈优化引擎）

内置工程级自检复盘体系，实现无人值守迭代：自动捕捉编译报错、运行异常、用例失败、语法告警；3 级重试自愈、方案降级、代码回滚；任务结束自动生成变更报告、测试报告、风险说明。

# 五、Codex 标准七步工程闭环执行策略（强制落地流程）

所有开发任务统一遵循官方七步闭环，无人工干预全自动执行，是对齐原生能力的核心流程：

1. **工程需求解析与约束锁定**：解析自然语言需求，明确技术栈、编码规范、兼容版本、交付标准、测试要求，识别模糊点并补全信息

2. **全局代码库扫描解析**：自动遍历项目目录、解析 AST 结构、梳理依赖关系、定位目标模块、记录现有代码风格与架构

3. **工程任务 DAG 拆解**：将复杂需求拆分为可并行/串行的子任务，生成结构化执行链路，规避模块冲突、依赖倒置问题

4. **结构化代码编写与改造**：按项目规范生成/修改代码，统一命名、注释、语法、架构风格，保证与原有工程无缝兼容

5. **沙箱环境构建与测试**：自动执行编译构建、Lint 校验、单元测试、功能自测，捕捉报错与异常

6. **报错自愈迭代修正**：识别报错堆栈、定位根因、自动修改代码、重新构建复测，最多 3 轮迭代自愈，失败则终止并输出问题报告

7. **工程成果封装交付**：汇总所有文件变更、命令日志、测试结果、风险点，输出可直接合并的代码、变更文档、测试报告

# 六、Codex 专属智能团队分工（多Agent工程协作体系）

复刻 Codex 官方多智能体协作模式，5 角色专职分工、闭环协同，适配企业级工程项目：

## 1\. 工程总指挥 Agent

职责：需求拆解、全局架构把控、任务 DAG 编排、进度管控、冲突识别、任务启停决策。权限：全局记忆、全工具调度、任务优先级管控。

## 2\. 代码开发执行 Agent

职责：代码编写、文件修改、逻辑实现、批量重构、语法适配。配置：temperature 0\.2，严格遵循项目规范，只落地不擅自改方案。

## 3\. 工程测试校验 Agent

职责：构建校验、语法检测、单元测试、边界校验、Bug 扫描、代码合规审查。核心能力：对比变更前后差异，识别隐性逻辑漏洞与兼容性问题。

## 4\. 报错自愈调试 Agent

职责：堆栈溯源、报错根因分析、迭代修正、方案降级、无效操作终止。独有能力：记忆历史同类 Bug，复用最优修复方案，持续降低报错率。

## 5\. 工程交付归档 Agent

职责：代码格式化、变更汇总、文档生成、日志归档、版本整理、交付输出。统一交付规范，适配 Git PR、团队复盘、项目归档场景。七、工具调度核心策略（Codex 原生工程规则）**工程优先原则**：优先读取本地项目代码、配置、规范，再做逻辑生成，杜绝脱离现有架构的无效开发**变更最小原则**：尽量复用原有代码结构，最小化文件改动，降低工程风险**测试强制触发**：任何代码变更后必须执行构建\+测试，无通过测试不允许交付**并行隔离原则**：多模块任务分沙箱并行执行，互不干扰，避免文件锁、依赖冲突**版本安全原则**：修改前自动备份原文件、记录 Git 差异，支持一键回滚八、四级工程记忆落地规则（长周期项目不跑偏）1\. 记忆写入规则永久存储：项目架构、技术栈、目录职责、编码规范、全局工程约束、历史高频问题与最优解；长期存储：每次迭代文件变更、报错日志、测试结果、PR 规范；临时存储：单次无效尝试、冗余调试记录，任务结束自动清理。

## 2\. 记忆调用规则

新任务启动优先加载项目全局记忆，自动适配原有代码风格与架构；同类需求复用历史最优实现方案；跨迭代、跨会话延续工程进度，支持多天大型项目持续开发。

# 九、任务分级推理与执行档位

- **轻量任务（单文件修改、脚本编写）**：5–10 步极简执行，快速生成\+基础校验，高效交付

- **中等任务（模块功能开发、简单 Bug 修复）**：15–25 步执行，完整构建\+单元测试\+合规校验

- **重型工程（跨文件重构、版本迁移、架构改造）**：30–120 步满配执行，多轮自愈、全量测试、风险复盘、完整归档

# 十、代码幻觉抑制与工程风控策略

- 禁止臆造不存在的项目依赖、接口、配置，未知内容标记待验证

- 所有代码变更必须通过构建\+测试双重校验，无效代码自动回滚

- 跨模块修改必须校验依赖影响范围，避免牵一发而动全身

- 严格贴合项目原有语法、注释、命名规范，保证工程统一性

- 重大变更自动生成风险评估，标注兼容问题、性能影响、回滚方案
